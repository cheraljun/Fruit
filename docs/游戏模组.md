# 游戏模组开发规范

## 定义

游戏模组（GameMod）是category为'gamemod'的插件，扩展游戏机制（时间、库存、战斗等）

## 核心架构

**Runtime集成** - 注册自定义函数到RuntimePlugin
**Blockly扩展** - 通过钩子注册积木块和代码生成器
**变量约定** - 定义所需变量（命名/类型/初始值）
**文档完备** - getDocs()提供使用说明和示例

## 代码结构

```typescript
export class XxxPlugin extends PluginBase {
  metadata = { id: 'gamemod.xxx', category: 'gamemod', requires: ['basicmod.runtime'] };
  
  protected async onInstall() {
    const rt = this.system.getPlugin('basicmod.runtime')?.plugin as RuntimePlugin;
    rt.registerFunction('funcName', this.funcImpl.bind(this));
  }
  
  hooks = {
    'blockly:register-blocks': (b: any[]) => [...b, ...自定义块],
    'blockly:register-generators': (g: Record<string, any>) => ({...g, ...生成器})
  };
  
  static getDocs(): GameModDocs { return {...}; }
}
```

## Blockly规范

**命名** - 前缀使用模组简称（time_add/inv_get）
**生成器** - 调用runtime.vars和已注册函数，不直接操作状态
**工具箱** - 模组名作为分类，colour统一45

## Runtime规范

**函数注册** - onInstall中调用registerFunction()
**变量访问** - 通过runtime.vars读写，不直接访问
**容错** - 变量不存在时返回默认值，不抛错

## 文档规范

getDocs()必须包含setup/scriptUsage/examples，textUsage可选
setup说明所需变量（英文名/类型/初始值）

## 目录约定

```
shared/plugins/gamemods/
  TimeSystemPlugin.ts

  index.ts
```

## 启用流程

插件商店启用 → 侧边栏显示文档 → 注入变量按钮 → Blockly工具箱自动添加积木块
