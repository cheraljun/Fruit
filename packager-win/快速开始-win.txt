mo 桌面版打包工具 - 快速开始

第一步：安装依赖

在 packager-win 目录下运行：

    npm install

（只需运行一次）


第二步：执行打包

    npm run build

等待 5-10 分钟，自动完成以下步骤：
  - 构建所有模块（backend, frontend, player-standalone）
  - 下载 Node.js 便携版
  - 复制文件
  - 安装依赖
  - 生成启动器
  - 打包成 ZIP


第三步：获取产物

打包完成后，在 output/ 目录下会生成：

    mo-Desktop-{时间戳}.zip

这就是可分发的桌面版。


分发给用户

用户使用方法：
  1. 解压 ZIP 到任意位置
  2. 双击"双击启动.cmd"
  3. 浏览器自动打开编辑器
  4. 开始创作
  5. 关闭窗口停止服务


常见问题

Q: 打包失败？
A: 脚本会自动检查并构建所有模块，无需手动构建。
   如果失败，检查网络连接和依赖安装。

Q: 下载 Node.js 很慢？
A: 脚本已使用国内镜像（npmmirror.com），请耐心等待

Q: 如何测试打包结果？
A: 解压 output/ 里的 ZIP，双击启动脚本测试

Q: 如何清理打包文件？
A: 在项目根目录运行 python rm.py
   会自动清理所有构建产物和 packager-win 的临时文件
