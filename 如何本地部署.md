# 本地版安装指南

## 1. 下载并安装 Node.js

访问 nodejs.org，下载 Windows X64 版本的安装包，按提示安装。

## 2. 验证 Node.js 安装

按 `Win+R` 键，会弹出"运行"窗口。

在运行窗口中输入 `cmd`，然后按回车键。

会打开一个黑色的命令行窗口，输入 `node -v` 后回车，如果显示版本号（如 v18.x.x），说明安装成功。

## 3. 解压项目文件

下载 MoScript 压缩包，解压到任意位置（建议桌面或 D 盘根目录）。

## 4. 安装 shared 依赖

打开解压后的 MoScript 文件夹，双击进入 `shared` 文件夹。

**在文件资源管理器顶部有个地址栏**（显示当前路径的那一栏），**点击地址栏**，里面的路径会变成可编辑状态。

删除地址栏中的内容，输入 `cmd`，然后按回车键。这会在当前文件夹位置打开命令行窗口。

在命令行窗口中输入：
```
npm install
```

按回车，等待安装完成（会显示很多文字滚动，这是正常的）。安装完成后，关闭这个窗口。

## 5. 启动前端

回到 MoScript 文件夹（根目录，不要进入任何子文件夹）。

点击顶部地址栏，输入 `cmd`，按回车。

在打开的命令行窗口中输入：
```
cd player-standalone\terminal && npm install && cd .. && cd visual-novel && npm install && cd .. && npm run build && cd .. && cd frontend && npm install && npm run dev
```

按回车，等待一段时间。当看到类似以下内容时，说明前端启动成功：
```
➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

**这个窗口必须保持打开，不要关闭。最小化到后台即可。**

## 6. 启动后端

回到 MoScript 文件夹，双击进入 `backend` 文件夹。

点击顶部地址栏，输入 `cmd`，按回车。

在打开的命令行窗口中输入：
```
npm install && npm run dev
```

按回车，等待片刻。当看到 `Server running on port 3000` 时，说明后端启动成功。

**这个窗口也必须保持打开，不要关闭。最小化到后台即可。**

现在你应该有**两个命令行窗口**在后台运行（一个前端，一个后端）。

## 7. 访问编辑器

打开浏览器（推荐 Chrome 或 Edge），在地址栏输入：
```
http://localhost:5173/
```

按回车，即可看到编辑器界面。

## 8. 数据备份

你的所有数据保存在 `backend/userdata/` 文件夹中，注意定期复制这个文件夹备份到其他位置。

## 9. 版本更新

更新到新版本时：
1. 在新版本中按上述步骤操作到第 4 步
2. 删除新版本的 `backend/userdata/` 文件夹
3. 把旧版本的 `userdata` 文件夹复制到新版本的 `backend/` 目录下
4. 继续执行第 5-7 步

## 10. 后续启动

第一次安装需要执行完整的安装+构建流程，之后每次启动只需要：

**前端窗口（在 frontend 文件夹下）：**
```
npm run dev
```

**后端窗口（在 backend 文件夹下）：**
```
npm run dev
```

保持两个终端窗口打开即可。

**注意：** 如果运行了清理脚本，需要重新执行第 4-6 步的完整安装流程。

---

## 常见问题

**Q: 端口被占用怎么办？**  
A: 关闭占用 3000 或 5173 端口的程序，或修改配置文件中的端口号。

**Q: 启动失败？**  
A: 检查 Node.js 版本是否为 16 或以上，删除 `node_modules` 文件夹重新 `npm install`。

**Q: 数据丢失？**  
A: 数据在 `backend/userdata/` 中，只要这个文件夹在，数据就在。
