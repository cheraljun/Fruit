## 核心原则

1. **每次只写一个层级**，写完等待检查
2. **用标签组织节点**，便于过滤和管理
3. **用文字描写制造氛围**，不用数值
4. **选择要有trade-off**，没有最优解
5. **遵循命名规范**，保持结构清晰
